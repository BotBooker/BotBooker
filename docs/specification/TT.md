# Техническое задание на разработку сервиса

Разработать бизнес‑модель для автоматизированной системы бронирования с отдельными чатами по каждому типу объектов аренды.

Пользователи приходят в чат-интерфейс в телеграмм или любой другой мессенджер с целью посмотреть, выбрать, забронировать и оплатить объект аренды.

## Термины

1. аренда - оплаченное время владения объектом
2. бронирование - бесплатное, короткое время ожидания оплаты

## Основная возможность пользователя

* выбрать объект аренды:
  - по дате и времени
  - по объекту аренды, его свойствам
* изменить бронь:
  - дату, время и продолжительность бронирования
  - объект аренды
* отменить бронь
* настроить получение уведомления
  - статус бронирования
  - напоминания о событиях
* оплатить
* интеграция с календарём
* посмотреть историю

## Основная возможность операторов

* управление через веб-панель:
  - аналитика
  - объект аренды
  - аренда
  - пользователь (по идентификатору вкл/выкл доступ)

## Технические условия

* Асинхронная обработка запросов для высокой нагрузки
* Кэширование данных
* Быстрая валидация бронирований
* Резервное копирование в облачное хранилище
* Ролевая модель доступа и аудит изменений
* Экспорт открытых данных в Excel/PDF

## Общая архитектура

Сервис представляет собой мессенджер‑бот (Telegram/WhatsApp/др.), интегрированный с бэкенд‑системой и базой данных.
Взаимодействие идёт по схеме:

* пользователь -> бот -> бэкенд -> БД -> бэкенд -> бот -> пользователь
* оператор -> бот -> бэкенд -> БД -> бэкенд -> бот -> оператор
* оператор -> веб-панель -> бэкенд -> БД -> бэкенд -> веб-панель -> оператор
